# specs/ -- 個別仕様書ディレクトリ

## 命名規則
`NNN-機能名.md` 形式で作成する。

例:
- `001-earthquake-realtime.md`
- `002-sse-streaming.md`
- `003-push-notification.md`
- `004-i18n-16lang.md`
- `005-shelter-finder.md`

## 仕様書テンプレート

```markdown
# NNN-機能名

## ステータス
MUST / SHOULD / MAY

## 概要
機能の目的と背景

## 要件
### 機能要件
-

### 非機能要件
-

## 設計
### データフロー
### UI/UX
### API

## テスト方針
-

## 関連
- spec.md の該当機能
- plan.md の該当フェーズ
```
