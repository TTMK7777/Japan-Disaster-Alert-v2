## 2. 未対応の指摘事項（次セッションの候補）

レビューで発見したが、挙動変更・API変更を伴うため**意図的に修正しなかった**もの:

### フロントエンド
- **デッドコード**: `VolcanoAlert.tsx` / `HazardMapLayer.tsx` / `AlertBanner.tsx` / `AlertBannerCompact` / `TsunamiLevelIndicator` はどこからも参照されていない。バックエンドAPI（volcanoes 等）に対応する実装済み機能のため、**接続するか削除するか要判断**
- `WarningBanner.tsx`: 5分ごとの自動更新で毎回スケルトンにフラッシュ（UX問題）
- `ThemeToggle.tsx`: aria-label が英語ハードコード（i18nキーは整備済み、props に language 追加が必要）
- nav の aria-label が ja/en のみ
- `usePushNotification.ts`: 通知許可拒否時に無言で return（呼び出し側に失敗が伝わらない）

### バックエンド
- `datetime.now()` が timezone-naive（レスポンス形式変更がフロントに影響しうる）
- `ContentSizeLimitMiddleware` が Content-Length のみ検査（chunked 転送をすり抜け）
- `PushSubscription.keys` の `p256dh`/`auth` が空文字で登録可能
- 各サービスの `_get_client`/`close` が5箇所重複（基底クラス化を提案）
